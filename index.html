[Cart] Total price shows £0.00 after applying discount code

Environment: Chrome 121.0.6167.85, Windows 11 Pro, Build: staging-2.4.1
Preconditions: User logged in, cart has 2+ items (tested with total £89.99)
Steps to Reproduce:
  1. Add any product to cart (I used "Wireless Headphones" £49.99)
  2. Go to cart page → /checkout/cart
  3. Enter discount code "SAVE10" in promo field
  4. Click "Apply" button
  5. Observe the total price field
Expected Result:

Total shows £80.99 (original £89.99 minus 10% discount)

Actual Result:

Total displays "£0.00". Discount shows "-£89.99" (full amount instead of 10%). Screenshot attached.

Severity: Critical Priority: High
2 / 3

[Login] Error message not displayed for incorrect password

Environment: Firefox 122.0, Windows 10, Build: prod-2.3.8
Steps to Reproduce:
  1. Open login page → /auth/login
  2. Enter valid email: test.user@gmail.com
  3. Enter wrong password: "wrongpass123"
  4. Click "Sign In" button
Expected Result:

Red error message appears: "Invalid email or password"

Actual Result:

Page reloads with no error message. User has no feedback that login failed. Only console shows 401 error. Confusing UX - user keeps trying same password.

Note:

Reproduced 5/5 times. Works correctly on mobile app (v1.2.0).

Severity: High Priority: High
3 / 3

[UI] Product images not loading on slow connection (3G)

Environment: Chrome 121.0 (DevTools → Network: Slow 3G), Windows 11
Steps to Reproduce:
  1. Open DevTools (F12) → Network tab
  2. Set throttling to "Slow 3G"
  3. Navigate to category page → /products/electronics
  4. Wait for page to load (approx 8-10 sec)
  5. Observe product grid
Expected Result:

Images load with placeholder/skeleton, then actual images appear

Actual Result:

Broken image icons shown for 6 out of 12 products. No lazy loading implemented. Images timeout after ~5 sec with no retry.

Suggestion:

Add loading="lazy" attribute and placeholder images for better UX on slow connections.

Severity: Medium Priority: Low
